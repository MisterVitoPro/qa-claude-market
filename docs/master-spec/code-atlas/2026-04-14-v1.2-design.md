# Code Atlas v1.2 Design

**Date:** 2026-04-14
**Status:** Draft (awaiting user review)
**Scope:** Fix dependency correctness bug (A), refactor hook to context-priming only (C), introduce `.code-atlas/` JSON cache artifacts (D), gitignore hygiene.

---

## 1. Motivation

Three problems with code-atlas v1.1:

1. **Dependency analysis is incorrect at scale.** The `atlas-dependencies` agent receives only the imports extracted from a 3-file × 100-line per-directory sample. Reported importer counts are a fraction of reality on any repo with more than a few dozen source files.
2. **The session-start hook duplicates update-skill logic and silently auto-writes to CLAUDE.md.** It forks thresholds from the update skill (drift risk) and mutates a tracked file without user consent.
3. **No persistent structured artifact exists.** Every incremental update has to parse the markdown section in CLAUDE.md to infer previous state. There is no cross-session cache, so `/update` cannot do real file-level diffing.

v1.2 addresses all three by moving the architecture index out of CLAUDE.md entirely, into a dedicated `.code-atlas/` cache directory that Claude loads at session start via the hook.

## 2. Goals

- Produce accurate dependency and importer-count data by extracting imports from every source file, not a sample.
- Make `/code-atlas:update` truly incremental via content-hash diffing that survives rebases and branch switches.
- Prime Claude with the full architecture index at session start at a bounded token cost (< 1.5 k tokens).
- Eliminate hook logic duplication; hook becomes read-only.
- Keep the cache out of version control so it does not bloat git history.

## 3. Non-Goals

- Changing what the three analysis agents produce semantically. Their output schemas stay the same; only the *input* to `atlas-dependencies` changes (full import graph instead of samples).
- Monorepo-aware sub-indexing (separate proposal).
- Drift detection against manual edits (no longer relevant once we stop writing into CLAUDE.md).
- Preserving the legacy CLAUDE.md architecture section (v1.x output). Users delete it manually after upgrading.

## 4. Architecture

### 4.1 File layout

```
.code-atlas/
  atlas.json      # curated summary, loaded by hook into session context
  state.json      # internal cache: file hashes, full import graph, raw agent output
```

The plugin writes only to this directory and to `.gitignore` (once, on first map). **CLAUDE.md is never modified by v1.2.**

### 4.2 Responsibilities

| File | Purpose | Size target | Consumer |
|------|---------|-------------|----------|
| `atlas.json` | Context primer for Claude: directory map, key files, tech stack, top high-traffic modules, dependency flow, build commands | ≤ 5 KB | Session-start hook (read-only) |
| `state.json` | Operational cache: per-file hashes, full import graph, full importer counts, raw agent JSON, schema/plugin version, baseline commit | Unbounded | `/code-atlas:update` (read + write) |

The separation guarantees session-start cost stays bounded even as repos grow.

### 4.3 Data flow

```
/code-atlas:map      -> writes atlas.json + state.json, appends to .gitignore
/code-atlas:update   -> reads state.json, writes both files
session-start hook   -> reads atlas.json only, prints to session context
```

## 5. Schema

### 5.1 Shared header

Both files begin with:

```json
{
  "schema_version": 1,
  "plugin_version": "1.2.0",
  "generated_at": "2026-04-14T15:30:00Z",
  "baseline_commit": "3f1b72e",
  "scan_root": "."
}
```

Schema mismatch on `/update` invalidates the cache and triggers a full re-scan.

### 5.2 `atlas.json`

```json
{
  "_header": { ... },
  "tech_stack": {
    "languages": ["TypeScript", "Python"],
    "framework": "Next.js 14",
    "build": "pnpm",
    "test": "vitest",
    "lint": "eslint + prettier",
    "ci": "GitHub Actions",
    "package_manager": "pnpm"
  },
  "architecture_pattern": "Layered MVC",
  "directory_map": [
    { "path": "src/api", "purpose": "REST handlers, one file per resource", "category": "source" }
  ],
  "key_files": [
    { "path": "src/index.ts", "role": "entry_point", "description": "Server bootstrap and middleware registration" }
  ],
  "high_traffic": [
    { "path": "src/utils/logger.ts", "importer_count": 47 }
  ],
  "dependency_flow": "routes -> controllers -> services -> models",
  "module_boundaries": [
    { "path": "src/auth", "type": "feature", "description": "Self-contained auth module" }
  ],
  "conventions": [
    { "area": "naming", "rule": "camelCase files, PascalCase classes" }
  ],
  "build_commands": [
    { "command": "pnpm dev", "purpose": "Start dev server with HMR" }
  ]
}
```

Caps (enforced by synthesis to keep size bounded):

- `directory_map` ≤ 30 entries. If the tree has more directories, collapse leaf directories into their parents with a "N subdirectories" note.
- `key_files` ≤ 15
- `high_traffic` ≤ 10
- `module_boundaries` ≤ 8
- `conventions` ≤ 10
- `build_commands` ≤ 8

Entries omitted from `atlas.json` due to caps still exist in `state.json`.

### 5.3 `state.json`

```json
{
  "_header": { ... },
  "file_index": {
    "src/api/users.ts": {
      "hash": "9f2c1a...",
      "size_bytes": 2341,
      "lang": "ts",
      "category": "source"
    }
  },
  "import_graph": {
    "src/api/users.ts": [
      "src/services/user-service.ts",
      "express",
      "src/utils/logger.ts"
    ]
  },
  "importer_counts": {
    "src/utils/logger.ts": 47,
    "src/services/user-service.ts": 12
  },
  "external_dependencies": [
    { "name": "express", "purpose": "HTTP framework", "usage": "core" }
  ],
  "circular_dependencies": [],
  "raw_agent_outputs": {
    "atlas_structure": { ... },
    "atlas_patterns": { ... },
    "atlas_dependencies": { ... }
  },
  "last_run": {
    "strategy": "full_scan",
    "duration_seconds": 42,
    "agents_used": 3,
    "files_scanned": 387,
    "files_hashed": 387
  }
}
```

**Hashing strategy:**

- For git-tracked files: use the git blob OID from `git ls-files -s`. Free - no file reads needed.
- For untracked files present in the working tree and not excluded: compute SHA-256 of content.
- The `hash` field does not distinguish between the two sources; both serve the same purpose (equality check for change detection).

**Import graph normalization:**

- Internal imports resolved to a repo-relative path (e.g., `src/utils/logger.ts`).
- External imports kept as the bare package name (e.g., `express`, not `node_modules/express/index.js`).
- Relative imports resolved by joining with the importing file's directory, then normalizing.
- Unresolvable imports (e.g., path aliases without tsconfig) kept as-is.

## 6. Workflow

### 6.1 `/code-atlas:map`

1. **File index build.** Run `git ls-files -s` to capture tracked files with blob OIDs. Walk the working tree for untracked files, respecting the exclusion list. Populate `file_index`.
2. **Config read.** Parallel Read of the existing config set (package.json, tsconfig.json, Cargo.toml, etc.). Unchanged from v1.1.
3. **Full-tree import extraction (fix A).** For every source file in `file_index`, run a regex pass appropriate to its language to extract import statements. Build `import_graph` and derive `importer_counts`. This is the new mandatory step.
4. **Informed sampling.** Select files for pattern-detection samples using the importer counts:
   - Top 10 files by importer count (global, not per-directory).
   - Plus up to 2 files per directory chosen for extension diversity (one file per extension, largest by size).
   - Read the first 150 lines of each selected file.
   Replaces v1.1's arbitrary "first 3 files per directory."
5. **Parallel agent dispatch.**
   - `atlas-structure`: receives full file tree + key file contents + categorization.
   - `atlas-patterns`: receives the informed sample + all configs + lightweight convention pre-pass statistics (file-count-based camelCase vs snake_case ratios, etc.).
   - `atlas-dependencies`: receives the **full** `import_graph` and `importer_counts` (not samples). Produces interpretation on top of accurate data.
6. **Synthesis.** Assemble `atlas.json` (applying caps from 5.2) and `state.json` from agent outputs and the raw data computed in steps 1-3.
7. **Write both files** to `.code-atlas/`. Create the directory if needed.
8. **Gitignore update.** If a `.gitignore` exists at the repo root and does not contain a line matching `.code-atlas/`, append `.code-atlas/` on its own line. If no `.gitignore` exists, do nothing.
9. **Migration notice.** If CLAUDE.md exists and contains `<!-- code-atlas:start -->`, print once:
   ```
   Legacy code-atlas section detected in CLAUDE.md (from v1.x).
   It is no longer maintained. You can delete the section between
   <!-- code-atlas:start --> and <!-- code-atlas:end --> manually.
   ```

### 6.2 `/code-atlas:update`

1. **Load baseline.** Read `state.json`. If missing, print a suggestion to run `/code-atlas:map` and stop. If `schema_version` does not match the current plugin's schema, run the full `/map` pipeline.
2. **Compute current index.** Same process as 6.1 step 1. Produces the current `file_index`.
3. **Hash diff.** Compare old vs new `file_index`:
   - `added`: paths in new not in old
   - `deleted`: paths in old not in new
   - `changed`: paths in both where hash differs
   - `renamed`: detected when a delete and an add have identical hashes (optional heuristic; if not implemented, counts as one delete + one add)
4. **Strategy selection.**

   | Strategy | Conditions | Work |
   |----------|-----------|------|
   | No-op | No diff, and current HEAD equals baseline commit | Print "up to date," exit |
   | Micro | Fewer than 5 changed source files AND no added/removed top-level source directories AND `full` arg not passed | Re-extract imports for changed files only; recompute affected importer counts; update `directory_map` deltas; update both files. No agent runs. |
   | Targeted | 5 to 30 changed files (inclusive) OR exactly 1-2 new top-level directories | Micro work + run `atlas-structure` agent scoped to changed areas; merge into state. |
   | Full | More than 30 changes, 3+ new top-level directories, malformed baseline, `full` arg, or `schema_version` mismatch | Run full `/map` pipeline. |

   Evaluate in order; first matching row wins.

5. **Write both files.** Update `_header.baseline_commit`, `_header.generated_at`, `last_run` block.

The hash-based diff makes update resilient to rebases and branch switches - v1.1 breaks when the stored commit hash no longer exists in history.

### 6.3 Session-start hook

```
1. If .code-atlas/atlas.json does not exist:
     Print: "No architecture index. Run /code-atlas:map for faster navigation."
     STOP
2. Read .code-atlas/atlas.json.
3. Get current HEAD: git rev-parse --short HEAD (silently skip the staleness block on error).
4. Print to session context:
     ## Code Atlas Architecture Index
     Cached: <baseline_commit>   HEAD: <current>
     Consult before using Explore or broad searches.

     <atlas.json contents, raw JSON>

     [If baseline_commit != current HEAD:
        Index is N commits behind. Run /code-atlas:update to refresh.]
5. Done.
```

Hook **never** writes, launches agents, runs diffs, or modifies any file. Target runtime: < 500 ms (two reads plus one `git rev-parse`).

## 7. Versioning & migration

- Plugin version bumps to `1.2.0`.
- `schema_version` starts at `1`. Future breaking changes to the JSON shape increment this; on mismatch, `/update` falls back to a full `/map`.
- Users upgrading from v1.1 to v1.2:
  1. First session after upgrade: no `.code-atlas/` exists, hook prints "run `/code-atlas:map`."
  2. User runs `/code-atlas:map` → writes new files, prints legacy-section notice if applicable.
  3. User optionally deletes the legacy section from CLAUDE.md.
- No automated migration of the legacy section. The v1.x content is not machine-parsed to seed v1.2 state; v1.2 starts from a clean scan.

## 8. Risks & tradeoffs

1. **Always-load hook token cost.** `atlas.json` is capped at ~5 KB (~1.2 k tokens) but loads on every session. Negligible cost for the benefit; accepted.
2. **Regex-based import extraction is approximate.** Misses exotic syntax (dynamic `import()`, runtime `require(variable)`, macro-generated imports). Accepted: still far more accurate than the v1.1 sample. Agents receive the caveat in their prompt.
3. **Hash-only diff misses semantically-equivalent moves.** A file moved without content changes keeps its hash, appears as delete+add. The optional rename heuristic (matching deleted hashes to added hashes) reduces noise but is not required for correctness.
4. **Cap-based trimming in `atlas.json` can hide relevant items on huge repos.** Mitigation: the caps are tunable via future configuration; full data available in `state.json` for deeper queries.
5. **Legacy CLAUDE.md sections go stale.** Accepted per user decision; print notice, let user delete.

## 9. Out of scope (deferred)

Captured here to close the ticket:

- Monorepo / workspace sub-indexing (pnpm/yarn/cargo/go workspaces).
- File-role weighting in update strategy (deleting an entry point ≠ deleting a doc).
- Shared scaffolding between qa-swarm and code-atlas for parallel-agent dispatch.
- JSON schema validation of agent outputs before synthesis.
- Per-language file-role heuristics beyond extension.
